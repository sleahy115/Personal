Digging into Git

This past week I attended a great workshop on Git hosted by PDXWIT(Portland women in tech). The workshop was a great beginner course in using git, but it made me curious about just how much I could do with git. In my research, I found that the commands are split into two categories, high level or porcelin and low level or plumbing commands. I was familiar with many of the high level commands like git add, git commit, and git clone. There were some I didn't know about however like git gui. I also spent some time getting more familiar with git rebase/ git revert/ and git reset. I admit that I have had issues with commits in the past and it is always good to know how to roll something back. Briefly, the difference between these commands is how they affect the history and when in the process you use them. All of these commands will help you go back to a previous commit but if you have already pushed to your remote repository you should use git revert. This is basically saying undo this commit both locally and remotely. If you have not pushed yet and you just need to change a local commit you should use git reset. For more information about which commands to use check this stack overflow thread Another favorite porcelain command is git whatchanged which does exactly what it sounds like and shows you what changed with each commit.
